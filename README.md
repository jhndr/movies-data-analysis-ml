# 🤖 IMDb Rating Prediction — AI/ML Project

A machine learning project that predicts IMDb ratings for movies and TV shows using three supervised regression models, with an interactive Streamlit dashboard for visualization and live predictions.

## 🎯 Objective

Predict the **IMDb rating** (a continuous value from 1.0 to 10.0) of a movie or TV show based on its objective attributes — release year, genre, number of votes, runtime, and gross revenue.

This is a **supervised regression** task. Three models are trained, evaluated, and compared:

| Model | Type | Library |
|---|---|---|
| Linear Regression | Baseline linear model | scikit-learn |
| Random Forest Regressor | Ensemble (100 decision trees) | scikit-learn |
| Neural Network (MLP) | Multi-layer perceptron (128→64→32) | scikit-learn |

---

## 📊 Dataset

The dataset used is `cleaned_movies.csv`, the preprocessed output of the data cleaning activity, derived from `movies_csv.xlsx` (IMDb movies and TV shows).

| Attribute | Detail |
|---|---|
| Source | movies_csv.xlsx (IMDb) |
| Records used | 6,252 (after dropping incomplete rows) |
| Target variable | `RATING` — IMDb score (1.1 – 9.4) |
| Train / Test split | 80% / 20% (random seed = 42) |

### Features used for training

| Feature | Description | Preprocessing |
|---|---|---|
| `YEAR` | Release year | Used as-is |
| `GENRE_ENC` | Primary genre | Label-encoded to integer |
| `LOG_VOTES` | Number of user votes | log(1 + votes) to reduce skew |
| `RunTime` | Duration in minutes | Used as-is |
| `LOG_GROSS` | Gross revenue in $M | log(1 + gross) to handle zero values |

## 🚀 How to Run

### Option A — Run the ML training script

Trains all three models, prints evaluation metrics, and saves 5 chart PNG files.

```bash
python ml_model.py
```

**Output:**
- Printed metrics table (MAE, RMSE, R²) for all three models
- Feature importance scores from Random Forest
- 5 chart PNG files saved in the same folder

### Option B — Run the Streamlit dashboard

Launches an interactive web app in your browser with data exploration, model results, and a live rating predictor.

```bash
streamlit run app.py
```

Then open your browser at: **http://localhost:8501**

> Make sure `cleaned_movies.csv` is in the same folder before running either script.

## 📈 Results

| Model | MAE | RMSE | R² |
|---|---|---|---|
| Linear Regression | 0.9052 | 1.1440 | 0.0227 |
| **Random Forest** ⭐ | **0.6603** | **0.9260** | **0.3597** |
| Neural Network (MLP) | 0.7130 | 0.9637 | 0.3065 |

**Random Forest achieved the best performance** across all three metrics, explaining ~36% of the variance in IMDb ratings using only five objective features.

### Feature Importance (Random Forest)

| Rank | Feature | Importance |
|---|---|---|
| 1 | Runtime | 0.386 |
| 2 | Log Votes | 0.309 |
| 3 | Genre | 0.201 |
| 4 | Year | 0.095 |
| 5 | Log Gross | 0.010 |

Runtime and number of votes together account for ~69% of the model's predictive power.

## 🧠 Key Findings

- **Random Forest outperforms** Linear Regression and Neural Network on this dataset
- **Linear Regression's near-zero R²** (0.0227) confirms the relationship between features and rating is non-linear
- **Runtime is the most important feature**, likely because content format (film vs. TV episode) correlates with runtime and rating patterns
- **Gross revenue has negligible impact** on predicted rating — commercial success does not reliably predict audience satisfaction
- The moderate R² of 0.36 reflects the inherent subjectivity of IMDb ratings, which are influenced by many factors not captured in structured data (e.g., cultural context, cast, marketing)

## 📝 Notes on TensorFlow

The activity specification mentions TensorFlow/Keras. The Neural Network in this project is implemented using **scikit-learn's MLPRegressor**, which provides equivalent multi-layer perceptron functionality. A TensorFlow/Keras equivalent would look like this:

```python
import tensorflow as tf

model = tf.keras.Sequential([
    tf.keras.layers.Dense(128, activation='relu', input_shape=(5,)),
    tf.keras.layers.Dense(64,  activation='relu'),
    tf.keras.layers.Dense(32,  activation='relu'),
    tf.keras.layers.Dense(1)
])

model.compile(optimizer='adam', loss='mse', metrics=['mae'])
model.fit(X_train_scaled, y_train, epochs=100, validation_split=0.1,
          callbacks=[tf.keras.callbacks.EarlyStopping(patience=10)])
```
